# FMCW Model

## Goal

Model FMCW LiDAR beat signals generated by coherent returns from one or more scatterers.

## Parameters

`FMCWChirp` must include:

- `center_wavelength_m`
- `center_frequency_hz`
- `bandwidth_hz`
- `sweep_time_s`
- `sample_rate_hz`
- `num_samples`
- `sweep_direction`
- optional `k_linearity_error`
- optional `laser_phase_noise`
- optional `linewidth_hz`

Derived:

```text
chirp_rate_hz_per_s = bandwidth_hz / sweep_time_s
```

## Range to Beat Frequency

For target range `R`:

```text
τ = 2R / c
f_b = chirp_rate * τ = chirp_rate * 2R / c
```

## Complex Beat Signal

For one scatterer:

```text
s_i(t) = A_i exp(j(2π f_bi t + φ_i))
```

For many scatterers:

```text
s(t) = Σ_i A_i exp(j(2π f_bi t + φ_i))
```

## Range Conversion

Given FFT bin beat frequency `f_b`:

```text
R = c * f_b / (2 * chirp_rate)
```

## Initial Approximation

- Monostatic transmitter/receiver
- One chirp per pixel
- Scanner state frozen during one chirp
- Complex baseband beat signal

## Future Extensions

- Up/down chirp velocity estimation
- Doppler shift
- scanner motion during chirp
- k-linearity error
- laser phase noise
- linewidth-induced phase noise
- IQ imbalance
