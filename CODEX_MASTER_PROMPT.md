# Codex Master Prompt

You are a coding agent developing a Python-based Coherent FMCW LiDAR Forward Simulator.

The project is not a pure ray tracing engine.

Ray tracing is only used for:

1. surface visibility
2. occlusion
3. mirror/scanner steering geometry
4. STL mesh intersection
5. visible surface patch detection

The actual FMCW LiDAR signal must be generated by coherent electric field summation over surface scatterers.

## Core Goal

Build a simulator that can model:

- Gaussian beam, M², divergence, waist, elliptical beam, line beam
- optical system with free space, lens, aperture, mirror, beam expander, scanner mirror
- actual scanner dynamics and time-dependent beam steering
- beam footprint movement on target surfaces
- STL/CAD-based geometry
- mirror, car paint, glass, tire, license plate, retroreflector material models
- rough surface scatterer map
- coherent speckle
- FMCW beat signal
- receiver aperture and coherent detection
- FFT/CZT processing
- range image, intensity image, speckle map, point cloud
- optional GPU acceleration through backend abstraction

## Fundamental Physics Rules

Do not sum powers when modeling speckle.

Correct:

```text
E_rx = Σ A_i exp(jφ_i)
P_rx = |E_rx|²
```

Incorrect:

```text
P_rx = Σ P_i
```

Power and field amplitude must be separated:

```text
P ∝ |E|²
E ∝ sqrt(P)
```

STL triangles are geometry objects, not optical scatterers.

Optical scatterers must be sampled separately on surface patches.

A fixed surface scatterer map must be reused across scan positions.

Scanner motion moves the beam footprint over the fixed scatterer map.

Do not regenerate random phases per pixel.
